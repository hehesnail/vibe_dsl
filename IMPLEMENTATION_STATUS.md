## 已完成工作

### 1. CodeGenBlackhole 重写 (src/target/codegen_blackhole.cc)

- **目标**: 生成纯 TT-Metal 格式的 kernel 代码，不含 TVM 头文件
- **实现**:
  - 新增 `GetKernelCode()` 方法，返回不含 TVM headers 的纯净 kernel 代码
  - 新增 `GenerateCopyKernelMain()` 方法，生成完整的 Copy Kernel
  - Kernel 包含正确的 TT-Metal 头文件：`dataflow_api.h`
  - 实现 CB (Circular Buffer) + NOC (Network on Chip) 操作
  - 支持 64-bit 地址传递（拆分为 lo/hi 32-bit）

**生成的 Kernel 代码示例**:
```cpp
// TT-Metal kernel generated by TileLang
#include <cstdint>
#include <cmath>

#include "dataflow_api.h"

void kernel_main() {
    // Get runtime arguments
    uint32_t src_addr_lo = get_arg_val<uint32_t>(0);
    uint32_t src_addr_hi = get_arg_val<uint32_t>(1);
    uint64_t src_addr = (static_cast<uint64_t>(src_addr_hi) << 32) | src_addr_lo;
    // ... CB reserve, NOC read/write, CB push/pop
}
```

## 已解决问题

### ✅ TT-Metal JIT 编译头文件路径问题 (已修复)

**问题描述**:
```
fatal error: dataflow_api.h: No such file or directory
```

**根本原因**:
- CodeGen 使用了错误的路径 `"dataflow_api.h"`
- TT-Metal 示例使用的是 `"api/dataflow/dataflow_api.h"`

**解决方案** (commit 260a652):
1. 更新所有 include 路径为 `"api/dataflow/dataflow_api.h"`
2. 在 `AddFunction()` 中清理 `decl_stream` 以移除 TVM 头文件
3. 修复 `GetKernelCode()` 以正确组合 `decl_stream` 和 `stream`

**验证结果**:
```
✓ Correct include path found: api/dataflow/dataflow_api.h
✓ No TVM headers found (correct)
```
